## 2. 開発・コーディングの絶対ルール 🛡️（※忘れたらあんじゅに怒られるぞ！）

1.  **安全確認の徹底（最優先！！）**:
    - コードを変更したら、**必ず `npm run build` を実行してエラーがないか確認すること。**
    - **「これくらい大丈夫」は禁止！** Vercel でのビルドエラーを防ぐため、プッシュ前のローカルビルドは絶対！
2.  **最新情報の調査・確認（最重要）**:
    - **ライブラリのバージョン、API 仕様、モデル名、リリース日などは、常に最新の情報を検索・確認してから作業すること。**
    - 自分の記憶や古いドキュメントに頼らず、現在の「最新」を追求すること。
3.  **こまめな保存とルール遵守**:
    - ビルドが通ったら、**必ず `git add .` -> `git commit` -> `git push` を行うこと。**
    - **【絶対遵守】シェルコマンド（特に git）を実行するときは、`&&` で繋げず、必ず 1 つずつ別々のツールコールに分けて実行すること。**（一度に繋げるとパースエラーになる可能性があるため！）
    - **【絶対遵守】コードの置換（replace）を行う際は、絶対に `// ...` や `(中略)` などと省略しないこと。常に完全なコンテキストとコードを提供すること。**
    - コミットメッセージは英語で、具体的かつ簡潔に。
4.  **コーディング規約**:
    - **日本語コメント**: コードの意図がわかるように、日本語で丁寧にコメントを残す。
    - **型安全**: `any` 型は禁止。`zod` や TypeScript の型定義をしっかり使う。
    - **TypeScript の型インポート**: `verbatimModuleSyntax` に対応するため、型のみのインポートには `import type { ... }` を使用すること。
5.  **ドキュメント駆動**:
    - 作業前・作業後に `docs/tasks.md` を確認・更新する。
6.  **絵文字の禁止とアイコンの使用**:
    - UI やコード内での絵文字の使用は禁止。
    - アイコンが必要な場合は、`lucide-react` などのライブラリを使用すること。
